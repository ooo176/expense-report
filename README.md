# 发票报销打印拼版工具

一个浏览器端的发票拼版打印工具。把出差报销时收到的 PDF / 图片发票一次性拖进来，自动排成每页 2 / 4 / 6 / 9 张（或任意行列），所见即所得地预览，再用浏览器原生打印对话框打到纸上。

整个项目就是一个 `expense-report.html`，没有依赖、没有后端、不上传任何数据，所有处理都在浏览器里完成。

## 特性

- 同时支持 **PDF 和图片**（JPG / PNG / WEBP / GIF / BMP），混合上传
- PDF 多页自动拆成单张，逐张参与排版
- 每页张数可选：**1×2 / 2×2 / 2×3 / 3×3**，或自定义任意行列
- A4 **纵向 / 横向**切换
- **拖拽排序**调整发票顺序，每张可单独旋转、删除
- **自动旋转贴合**：横版发票放进竖版单元格时自动旋转 90°，最大化利用纸面
- 间距可调（0–6mm），分割线可关
- 渲染质量切换：150 / 200 / 300 DPI
- **打印干净**：通过 `@media print` 隐藏所有控件，输出只有拼好的 A4 页

## 快速开始

```bash
# 直接用浏览器打开即可（推荐 Chrome / Edge / Safari 最新版）
open expense-report.html
```

或者通过任意静态服务器伺服（IDE 内置的也行），地址栏访问 `expense-report.html`。

### 用法

1. 把 PDF / 图片拖进上传区，或点击选择文件
2. 顶部工具栏选每页张数、纸张方向、间距、是否自动旋转
3. 在文件列表里拖拽缩略图调整顺序，单张可点 `↻` 旋转、`×` 移除
4. 下方实时显示打印预览，所见即所得
5. 点右上角「打印」，在系统打印对话框里：
   - 边距选 **「无」**
   - 取消勾选 **「页眉和页脚」**
   - 缩放选 **「默认」** 或 **「实际大小」**

## 技术细节

### 依赖

唯一外部依赖是 [PDF.js](https://github.com/mozilla/pdf.js) `4.7.76`（ESM 版本），通过 CDN 加载：

- `pdf.min.mjs` / `pdf.worker.min.mjs` 来自 cdnjs
- `cmaps/` 和 `standard_fonts/` 来自 jsdelivr 的 npm 镜像（cdnjs 不带这两个目录）

第一次加载需要联网拉这些资源，浏览器之后会缓存。如果想完全离线，把这几个文件下载到本地、改 `<script type="module">` 里的 import URL 即可。

### 为什么用 PDF.js 4.x

中国「全电发票」（2022 年后的电子发票）用的字体子集，PDF.js 3.x 渲染时会**丢字**——背景、印章、二维码都正常，唯独文字消失。4.x 对这类嵌入字体编码的支持显著改善。

### CMap 配置

CJK PDF（中文 / 日文 / 韩文）需要 cMap 字符映射数据，否则字符码点会落到错误的字形上。代码里通过 `cMapUrl` + `cMapPacked: true` + `standardFontDataUrl` 把这些资源指给 PDF.js。

### 透明背景处理

PDF 渲染前先把 canvas 整面填白，避免透明 PDF 转 JPEG 时透明区被填成黑色。

### 打印分辨率

`scale` 按目标 DPI 反推（默认 200 DPI），并 clamp 到长边 12000px 以内，避免触发浏览器 canvas 上限。PDF 优先输出 PNG（文字锐利），数据 URL > 8MB 时回落到 JPEG 0.95。

### 旋转贴合

每个单元格的 mm 尺寸由 JS 实时计算（A4 尺寸 - 内边距 - gap），需要旋转 90°/270° 时给图片包一层 wrapper 并交换其宽高，让旋转后的视觉框正好贴合单元格。

## 浏览器兼容性

- 使用了 ES Module，需要任意 2020 年后的现代浏览器
- 测试过：Chrome / Edge / Safari 最新版
- IE / 旧版 Edge：不支持

## 隐私说明

发票通常含个人 / 公司敏感信息。本工具：

- **完全在你本地浏览器内运行**，发票不会上传到任何服务器
- 没有任何分析、追踪、埋点
- 仅外部请求是从 CDN 加载 PDF.js 库本身（不发送任何 PDF 内容）
- 仓库的 `.gitignore` 默认排除了所有 `*.pdf` 和图片，避免不小心提交真实发票

## 已知限制

- 部分由 OFD 转 PDF 的发票若编码异常，文字可能仍渲染不完整 —— 这种情况下用「另存为图片」或截图后传 PNG 可绕过
- 单次处理几十张发票没问题，几百张时可能会因数据 URL 过大导致打印对话框卡顿，建议分批
- 浏览器原生打印不支持精确到点的色彩管理，不建议用本工具打印需要色彩还原的票据
